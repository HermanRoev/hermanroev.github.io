Accessibility for all

How we improved accessibility

Designing phase

While designing our page we had accessibility in mind. To start off with, we have the same navigation bar and footer on all of our pages (except from the "topic page" which is designed as an independent page). This was a decision we made not only for aesthetic reasons, but also for the user to get familiar with the format. To further the point of a familiar format we also used as much of the same layout code for our different pages as possible. This creates a similar flow on all pages which makes it easier for any user to get used to the page, with the added bonus of making it easier to make a change on all pages with the same line of code.
We also decided to have a lot of white space and text with high contrast to make everything easily visible and intelligible. The use of clickable buttons in the header section on most of our pages also makes it very easy for the user to navigate to different sections of the page.
To make structural hierarchy in our page we used different h1, and h2 elements, however this was later improved upon when using WAVE´s automated test. Another clear example of ease of accessibility on our page is on our “dining” page, where we used icons to make it fast and easy to see common allergens in different dishes.
A responsible site was also a big factor when we were coding. We had a goal to make all pages fully responsible for all screens and coded with this goal in mind. Everyone of our pages are now accessible and scalable to all screen sizes used as examples in the inspect element of our browser. This makes our website accessible and usable for most screen sizes.

WAVE automated test

After finishing all of our pages' visual design we ran the WAVE automated test tool on our page to check how we could improve our page for better accessibility. The most important alert we got was that the icon that open and close our navbar was not targetable for keyboard users. This essentially means that our navbar was inaccessible for keyboard users, which we fixed by placing the icon inside an button element. As earlier mentioned we also improved upon the structural layout using the WAVE tool by changing h elements to the correct order as well as using nav, main, section and footer tags instead of too many divs. These changes were the biggest and most important changes we made, but we also had to make several smaller changes such as for example giving the pages a set language using the html lang tag and giving all pages a title element.
